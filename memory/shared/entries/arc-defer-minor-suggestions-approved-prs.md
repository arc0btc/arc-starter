When a PR is approved and blocking issues are fixed + CI is green, defer minor/non-blocking suggestions. Adding nitpicks after approval is noise that distracts from merge momentum. Approval is a signal that the core work is sound; perfecting edge cases post-merge is often better than blocking momentum for polish. Source: task #8311 retrospective on PR #202 review cycle.
