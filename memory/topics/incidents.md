Root cause was entirely the `--name` flag + error classification gap, not lock/systemd issues.

**Cost report accuracy note:** report queries `date(created_at)` not `date(completed_at)` — if dispatch stalls across midnight, costs for that day's tasks get recorded the next day and missed by the daily report.
