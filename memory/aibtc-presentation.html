Overview

What is Arc?

The Core Idea

  • Autonomous agent running on a 24/7 loop
  • Everything is a task in a SQLite queue
  • Sensors detect signals, create tasks (no LLM)
  • Dispatch executes tasks via Claude Code subprocess
  • Skills are knowledge containers — scoped context per task
  • CLI-first: every action is an arc command

Built With

  • Bun runtime — fast, TypeScript-native
  • SQLite — task queue, cycle log, cost tracking
  • Claude Code — LLM dispatch (Opus, Sonnet, Haiku)
  • systemd — services for sensors + dispatch
  • Git — memory versioning, fleet sync
  • Stacks / Bitcoin — on-chain identity & signing

No LLM — pure TypeScript. Detect signals, create tasks.
↓ tasks table (SQLite) ↓
Dispatch (lock-gated, 30min max)

Why Build an Arc Agent?

  • Autonomous, not just reactive — sensors detect, dispatch acts, 24/7
  • Bitcoin-native identity — on-chain presence, cryptographic signing, verifiable actions
  • Composable skills — add capabilities without touching core code
  • Fleet-ready — one agent or five, same architecture
  • Cost-efficient — 3-tier model routing puts the right model on the right task
  • Open source — fork it, modify it, make it yours
