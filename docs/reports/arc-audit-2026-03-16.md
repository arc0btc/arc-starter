- Root cause: Starlight's `autogenerate` sidebar sorts alphabetically by filename, so date-prefixed posts appeared oldest-first
- Fix: Added `sidebar.order` frontmatter to all 20 posts (1=newest V2 Sprint, 20=oldest Hello World) — sidebar now shows reverse-chronological
- Added complete "All Posts" listing to `blog/index.mdx` with reverse-chronological order, monthly grouping (March/February 2026), and one-line descriptions per post
- Kept existing "Threads" card grid as curated entry points for new readers
- Build verified clean. PR: https://github.com/arc0btc/arc0me-site/pull/4

**Footer + verify signature investigation (task #5930):**
- **Footer `[arc0.btc](https://arc0.me)`:** Correct and intentional. `arc0.btc` is Arc's BNS name used as a byline/attribution, linking to the author's website. Not a bug — it follows a standard "signed by [identity]" pattern. Minor UX note: the link is self-referential (reader is already on arc0.me). Could be changed to `/about` for more utility, but cosmetic only.
- **Verify link signatures:** Working correctly on the live site. Verified via WebFetch on multiple posts (`three-of-three`, `v2-sprint`, `arc-starter`, `the-noise-floor`). All 20 posts have BTC (BIP-137) and STX (SIWS-compatible) signatures in frontmatter, correctly extracted into `/blog/{slug}.json` endpoints at build time. The `[slug].json.ts` endpoint uses `gray-matter` to parse frontmatter and maps `frontmatter.signatures` → `postData.signature` with both `btc` and `stx` sub-objects.
- **"Three Models, One Queue" reference:** No post with this title exists. Closest match is "Three of Three" (`2026-02-25-three-of-three`) — signatures verified present. The issue whoabuddy observed may have been a stale Cloudflare deployment that has since been updated by the `blog-deploy` sensor.
- **No code changes needed.** Both the footer format and verify endpoint are functioning as designed.
