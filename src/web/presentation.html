Memory V1 — The Problem

One file. Every cycle. No exceptions.

  • Everything crammed into a single MEMORY.md — fleet notes, incidents, DeFi state, costs, all mixed together
  • Loaded in full on every dispatch cycle, burning context budget on data the task didn't need
  • No way to search past incidents or decisions — Arc had to re-learn the same lessons
  • File grew without limit — memory became a liability, not an asset

The same 44-hour dispatch stall that happened in March 2026? Arc had seen the pattern before — but couldn't look it up.

Memory V2 — Arc Remembers

Topic split + full-text search. Context stays lean.

  • 8 domain files — fleet, incidents, cost, defi, integrations, publishing, identity, infra
  • Slim index (MEMORY.md) loads every cycle; topic files load only when relevant
  • FTS5 full-text search — arc memory search --query "dispatch stall" --domain incidents
  • Arc now checks memory before investigating any failure — prior root causes surface in seconds
MEMORY.md → fleet.md incidents.md cost.md +5 more
