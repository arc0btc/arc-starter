### What It Is

ByteRover is a native memory plugin for OpenClaw (an agentic coding assistant, similar to Claude Code). It ships as an NPM package and integrates directly into OpenClaw's ContextEngine.assemble() via a merged PR (#50848 on 2026-03-21).

Key claim: >92% retrieval accuracy on LoCoMo and LongMemEval benchmarks using a structured, file-based Context Tree — zero vector databases required.

### Architecture: The 24/7 Memory Loop

Three layers:

1. **Context Engine (Real-Time):**
   - Before each model run (Assemble): Injects curated knowledge directly into system prompt
   - After each turn (After Turn): Evaluates turn output, saves new decisions back to memory tree
   - Delegates memory compaction to OpenClaw (ownsCompaction: false) — non-invasive

2. **Automatic Memory Flush (Safety Net):**
   - Monitors context window token usage
   - Before token limit forces compaction, extracts + saves valuable insights from current session

3. **Daily Knowledge Mining (Morning Review):**
   - Native cron job fires at 9:00 AM daily
   - Sweeps prior day's memory logs, extracts architectural decisions and patterns
   - Persistent background debrief, no manual review required

## Summary

### Mission Relevance

HIGH — direct architectural intelligence for Arc's own memory system design.

ByteRover solves the same problem Arc's memory system solves: persistent stateful context across agent sessions.

Key differences from Arc's current approach:

| Dimension | Arc (current) | ByteRover |
|---|---|---|
| Storage | Flat MEMORY.md + git | Structured Context Tree (hierarchical file-based) |
| Retrieval | Full file load per dispatch | LLM-curated, query-aware assembly |
| Compaction | Manual / periodic consolidation | Automatic + daily mining cron |
| Integration | Dispatch system prompt injection | Native ContextEngine.assemble() hook |
| Accuracy | Not benchmarked | >92% (LoCoMo/LongMemEval) |

### Key Takeaways for Arc

1. **Context Tree > flat memory file.** Arc's MEMORY.md is flat. ByteRover's structured Context Tree with LLM-curated indexing achieves dramatically better retrieval. Worth evaluating if Arc's memory scaling hits quality problems.

2. **Query-aware assembly is the unlock.** The OpenClaw PR added optional prompt parameter to assemble() so retrieval is informed by the current user query. Arc's memory loads the full file regardless of task relevance. Selective injection based on task subject would be more token-efficient.

3. **Daily knowledge mining cron.** Arc does periodic consolidation triggered manually. ByteRover runs an automated daily sweep. This is a pattern worth adopting — a lightweight sensor that runs once/day to extract patterns from recent cycle logs.

4. **Automatic memory flush before context overflow.** Arc relies on a hard token limit. ByteRover actively monitors and flushes before overflow — proactive rather than reactive.

5. **30k downloads / 500k impressions** on prior memory skill = strong signal that structured agent memory is a high-demand product. Potential D1 revenue angle if Arc's memory architecture becomes a publishable pattern or tool.
