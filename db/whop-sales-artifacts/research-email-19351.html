1. Themes — the convergence across links

Four independent practitioners landed on the same conclusion in the same week, none citing each other:

  • The harness is commoditizing; the durable asset sits around the loop, not in it. Iusztin: "Models are commoditized. Harnesses are commoditized. The only moat is your context layer." Rohan Paul on the Claude Code paper: it works "not because of a complex AI brain but because a simple loop is surrounded by a huge system for tools, safety, memory, permissions, recovery." Akshay Pachaar: the self-compiling harness. Mayank: "eval-perfected harnesses." The model and the agent loop are not the differentiator. The scaffolding is.
  • The unit of capability is the small sharp skill; the frontier is composing many, not building bigger. Ibryam's top-10 star data: "small, sharp workflows are winning." Elvis on SkillWeaver: compositional routing decomposes a query, retrieves a skill per sub-task, composes a DAG plan across 2,209 real skills. Building one giant skill is the losing move.
  • Close the feedback loop — the output should be a better artifact, scored reproducibly. Zach Lloyd's self-improvement loop edits the agent's own Skills from feedback. Mayank publishes a concrete score: mechanical checks (objective) plus weighted LLM judge, pass at ≥0.7. The loop produces a better skill, measured.
  • Context is the scarce resource; compaction and indexing are first-class engineering. Trevin's RTK compacts verbose tool output at the hook layer (git status −60.9%). DataChaz indexes the codebase into an AST graph for token-cheap retrieval. Same problem from two angles.

The editorial read: the market is now naming the architecture Arc bet on a year ago. Task queue, git-versioned memory spine, sensors, escalation ladder — that is "own the context and feedback layer" before it had a name. The convergence is the signal: this is not a fad, it is where four serious builders independently arrived.
