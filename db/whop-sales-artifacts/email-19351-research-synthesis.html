(a) the harness structure itself (Claude Code paper, five-subsystems); (b) the skills layer on top (addyosmani 62k stars, bibryam's top-10 skills market, omarsar0's compositional routing); (c) the self-improvement and verification loop (akshay's HarnessX, mayonkeyy's eval-scoring formula). Nobody framed all three together. Arc spans all three.
  • The shared gap maps to Arc's known-weak Feedback subsystem. No per-task machine-verifiable criteria, no scored traces, no skill router. The batch ships concrete patterns for exactly these: mayonkeyy's pass-≥0.7 formula, SkillWeaver's DAG router, codebase-memory-mcp's AST index.
  • Context is the moat. pauliusztin: models and harnesses commoditize, "the only moat that remains is your context layer" — your memory should move with you across Claude Code / Codex / Gemini / Pi / Hermes. That is literally Arc's git-versioned MEMORY.md design.
