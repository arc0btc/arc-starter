## X Articles API Feasibility Research (task #6120)

### Verdict: Web-Only — No API Support

X Articles **cannot be created via the X API** as of March 2026.

### Findings
- X API v2 has no Articles endpoint. Only plain-text posts via POST /2/tweets (25k chars for Premium)
- Official X developer docs (docs.x.com) cover Posts, Users, Spaces, DMs, Lists, Trends — no Articles
- Developer community forum has an 'Articles API' feature request thread (devcommunity.x.com/t/articles-api/224208) — no X response or timeline
- Typefully explicitly documents that rich-text Articles require manual web editor paste — cannot be automated via API
- No public roadmap entry, no reverse-engineered endpoint, no timeline from X
- Browser automation workaround exists (Playwright) but violates X ToS

### What IS Available via API
- Plain longform post (25k chars) via POST /2/tweets — no bold/italic/headings/inline images
- This is what we're shipping in task #6119 (280→25k char limit fix)

### Recommendation
**Do not build an X Articles skill yet.** Rationale:
1. No API = no automation without ToS violation
2. Monitor dev forum thread for API announcement
3. If X ships Articles API: build x-articles skill with signal sensor + draft/post CLI
4. In the interim, the 25k plain-text upgrade (task #6119) covers most longform value

### Future Skill Sketch (when API ships)
- Skill: x-articles
- sensor.ts: detects high-signal blog posts or timely ecosystem events queued for article treatment
- cli.ts: arc skills run --name x-articles -- draft --blog-post-id <id>  / -- publish --draft-id <id>
- Integration: optional step in blog-x-syndication for high-priority posts
