## Tool Integration Analysis

### 1. DUAL STACKING TOOLS (Bitflow/Zero Authority)

**Current coverage:**
- stacks-stackspot: Lottery stacking on stackspot.app (auto-join pots, claim rewards)
- Minimal Bitflow/Zero Authority DAO stacking support

**v1.36.0 additions suggest:**
- Bitflow-specific stacking strategies
- Zero Authority DAO stacking integration  
- Advanced PoX delegation and pooling

**Integration recommendation:** CREATE NEW SKILL: `zero-authority-staking`
- Purpose: Autonomous participation in Zero Authority DAO stacking
- Sensor: Monitor DAO proposals, auto-delegate STX when profitable
- CLI: Propose stakes, claim rewards, query delegation status
- Keep stacks-stackspot for lottery (separate from DAO)

---

### 2. MEMPOOL MONITORING (transaction tracking)

**Current coverage:**
- mempool-watch: Bitcoin L1 mempool fee rates + Arc's address incoming txs
- 10min cadence, fee spike detection, unconfirmed tx monitoring

**v1.36.0 additions suggest:**
- Enhanced transaction tracking (multiple addresses)
- Advanced transaction status/replacement monitoring
- Pending transaction analysis

**Integration recommendation:** EXTEND `mempool-watch` skill
- Add: Multi-address transaction tracking
- Add: Replace-by-fee (RBF) monitoring  
- Add: CPFP (child-pays-for-parent) suggestions
- Cadence remains 10min

---

### 3. MESSAGE SIGNING TOOLS

**Current coverage:**
- bitcoin-wallet: BTC message signing (BIP-340/322), STX signing
- arc-reputation: BIP-322 signed peer reviews
- Code history: Nostr signing (NIP-01) already added in v1.28.0 (commit 18108de)

**v1.36.0 additions suggest:**
- Additional signing schemes/protocols
- Advanced signature verification
- Multi-format signing (unified interface)

**Integration recommendation:** CREATE NEW SKILL: `signing-suite`
- Purpose: Unified crypto signing across all protocols
- Features: BIP-322, SIP-018, Nostr NIP-01, ERC-191
- CLI: unified sign command + verify command
- Keep bitcoin-wallet for basic Bitcoin/STX operations
- Use signing-suite for advanced/cross-chain work
