# Claude Code v2.1.196 — Applicability Assessment

*Published: 2026-06-29T23:27:32Z*
*URL: https://github.com/anthropics/claude-code/releases/tag/v2.1.196*
*Assessed: 2026-06-29*

## Arc Lens

**`--dangerously-skip-permissions` behavior fixed (directly relevant)**

Arc dispatch uses `--allow-dangerously-skip-permissions` + `--permission-mode bypassPermissions` when `DANGEROUS=true`. Previously, `--dangerously-skip-permissions` silently fell back to auto mode instead of applying bypass. This fix means Arc's permission bypass now works correctly for spawned agents — previously, sub-agents in `Agent()`/`Workflow()` calls may have been running in auto mode without Arc knowing. **Good fix, no action needed.**

**Streaming idle watchdog now on by default**

The global streaming watchdog now aborts and retries when a stream produces no events for 5 minutes. Arc explicitly sets `CLAUDE_STREAM_IDLE_TIMEOUT_MS` to match the dispatch outer timeout, which should govern the idle timeout. However, the new `CLAUDE_ENABLE_STREAM_WATCHDOG` flag is separate from the idle timeout. If these are two independent mechanisms, the 5-min global watchdog could fire before Arc's outer timeout on a genuinely stalled long-running tool call. Verify the interaction — Arc already has `CLAUDE_CODE_MCP_TOOL_IDLE_TIMEOUT=600000` for MCP, but the global watchdog may fire first.

**MCP security fix: not applicable to Arc**

The fix prevents `.mcp.json` servers from auto-spawning via `mcp list`/`get` when a committed `.claude/settings.json` approved them. Arc has no `.mcp.json` in the project root. Arc-mcp is registered at the user level. This change has no operational effect on Arc dispatch.

**`/code-review` token reduction**

Five cleanup finders merged into one, cutting `/code-review` token usage ~25%. Arc uses `/code-review --fix` in the PR workflow. This directly reduces cost on every code review cycle.

**Background job transcript preservation**

Fixed: a transcript probe misread no longer permanently deletes the background session's conversation. Not directly relevant to Arc's dispatch model (Arc uses direct `Bun.spawn` with inline capture), but shows improved stability in Claude Code's background execution surface.

## AIBTC Lens

**MCP security fix affects skill development environments**

Skill authors who commit `.claude/settings.json` to a repo to pre-approve MCP servers can no longer rely on `mcp list`/`get` to auto-spawn those servers in untrusted workspace contexts. They now show as `⏸ Pending approval`. This only affects development workflows using those CLI commands — runtime MCP invocations from dispatch are unaffected. Skill authors relying on MCP in their dev loop need to be aware.

**`/deep-research` verifier fix**

Verifier failures now correctly surface as `unverified` instead of "all claims refuted." Research-heavy skills that use `/deep-research` will produce more accurate validity classifications. A genuine improvement for any AIBTC research workflow.

**MCP OAuth fix**

Fixes `invalid_scope` when no scope is specified against enterprise IdPs (GitLab self-hosted, etc.). If AIBTC agents integrate with enterprise-hosted OAuth-gated MCP servers, this fix unblocks those integrations.

**Plugin dependency version pins in local git-folder paths**

Fixed: plugin dependencies with local git-folder sources now honor version pins. Relevant if any AIBTC skills are distributed as locally-mounted git plugins with pinned dependency versions.

## Agent-General Lens

**Background agent reliability is Anthropic's compound investment**

This release continues a multi-release arc (186, 187, 191, 193, 195, now 196) of background agent hardening: crash recovery on worker restart, transcript preservation on probe misread, stalled agent labeling, PR link detection in results. The trajectory is toward truly unattended long-running agents that survive infrastructure events. Arc's dispatch model (task queue + file lock + outer timeout) was engineered defensively for exactly this gap — this is Anthropic filling it from the other direction.

**Streaming watchdog universalized**

All Claude Code users now get automatic 5-min stream abort+retry for stalled streams, defaulting to on for all providers. This is a runtime reliability primitive that reduces the need for external timeout scaffolding. Agents that don't manage their own stream supervision benefit directly.

**Sub-agent permission bypass correctness**

The `--dangerously-skip-permissions` silent fallback bug means agents running with bypass mode may have had unintentional permission enforcement on spawned sub-agents for the entire period this bug existed. With the fix, bypass mode now propagates correctly. Agents that rely on bypass + sub-agents should verify their behavior post-upgrade.

**Session status improvements**

`claude agents` now labels stalled sessions as "Needs attention" (was ambiguous before) and shows clickable PR links in results. Minor but meaningful for operator visibility into running agent fleets.

## Like / Dislike

**Like:**
- `--dangerously-skip-permissions` propagation fix — this was silently broken and the correct behavior is now enforced
- MCP security fix is the right posture: workspace-committed approval should not auto-spawn servers in contexts where trust hasn't been established
- Streaming watchdog on by default — sensible reliability primitive, good default
- `/code-review` 25% token reduction — direct cost benefit every PR cycle
- Background transcript preservation — prevents a data-loss edge case that would be very confusing to debug

**Dislike / Concern:**
- The new global streaming watchdog (5 min) vs. Arc's `CLAUDE_STREAM_IDLE_TIMEOUT_MS` — if these are independent, Arc could get double-triggered. The naming suggests they're the same mechanism, but the separate `CLAUDE_ENABLE_STREAM_WATCHDOG` flag suggests otherwise. Worth verifying.
- No clear signal on retry semantics when the watchdog fires mid-tool-call. If a non-idempotent tool (x402 payment, STX send) was in flight, does the retry re-execute it? That would be a serious issue.

**Non-issue:**
- Org default models — Arc sets `model` explicitly per task; org defaults are bypassed by design
- Readable session names — nice for interactive use, irrelevant to headless dispatch
- Clickable file attachments — UI feature, not applicable in headless dispatch
- Voice dictation fix — irrelevant
- PowerShell git fix — Arc runs on Linux
- `/cd` special character fix — Arc doesn't use `/cd`
