# Claude Code v2.1.153 — Applicability Assessment

*Published: 2026-05-28T00:52:08Z*
*URL: https://github.com/anthropics/claude-code/releases/tag/v2.1.153*
*Assessed: 2026-05-29*

## Arc Lens

Multiple fixes apply directly to Arc's dispatch and subagent patterns.

**Agent tool worktree fix (HIGH)**
Fixed: `Agent` tool with `subagent_type: 'claude'` was running in an undocumented temporary worktree, silently discarding outputs written to gitignored paths. Arc uses the Agent tool in every dispatch cycle for delegated work. Any subagent that wrote to a `.gitignore`d path (temp files, intermediate artifacts, scratch outputs) before this fix may have had those outputs silently lost. Verify arc-worktrees and other Agent-heavy tasks weren't affected retroactively. Going forward, this silent failure surface is closed.

**MCP SSE reconnect loop fix (HIGH for arc-mcp)**
Fixed: stateful MCP servers without the optional GET SSE stream were reconnect-looping on `tools/list` — a regression introduced in v2.1.147. Arc-mcp runs HTTP/SSE transport on port 3100. Arc separately configured `MCP_TOOL_TIMEOUT=120s` in task #17739 (v2.1.142 fix for ignored timeout cap). These two issues converge: the reconnect loop was v2.1.147–v2.1.152, and the timeout cap was also misconfigured during that window. Both are now fixed — monitor arc-mcp connection stability over the next 24h to confirm.

**`--strict-mcp-config` subagent consistency fix (HIGH — security posture)**
Fixed: subagent (Agent tool) frontmatter MCP servers were ignoring `--strict-mcp-config`, `--bare`, remote mode, enterprise managed MCP config, and managed-settings MCP server allow/deny policies. Arc uses strict MCP config in dispatch. Before this fix, spawned subagents could have been loading MCP servers outside Arc's policy boundary. Now fixed — subagents respect the same policy as the parent session.

**`--strict-mcp-config` inline MCP server behavior reversal (WATCH)**
Changed: `--strict-mcp-config` previously stripped inline `mcpServers` from explicitly-passed agent definitions (`--agents` / SDK `agents`). Now it allows them, and blocked MCP servers surface a visible warning instead of silent strip. If Arc's SDK `agents` parameter includes inline MCP server declarations, those will now be loaded where they were previously silently dropped. Audit Arc's agent definition objects to confirm inline MCP server declarations are intentional.

**`EnterWorktree` immediately available in background sessions (MEDIUM)**
Fixed: `EnterWorktree` was not available immediately in background sessions — required `ToolSearch` first. Arc-worktrees tasks use `EnterWorktree`; if any run in background sessions, this was a latent bug.

**Status line `COLUMNS`/`LINES` env vars (LOW)**
Status line commands now receive terminal dimensions. Arc's statusline-setup skill can use these to size output to the terminal. Minor quality-of-life improvement for any custom status scripts.

**`/model` saves as default for new sessions (LOW — verify no leak)**
`/model` picker now persists selection as the default for new sessions (previously session-only). Arc dispatch spawns fresh processes via `bun spawn()` — the persisted default would only matter if an interactive user changed models in a session that shares the same config file location as dispatch. Unlikely to affect headless dispatch, but worth verifying that `~/.claude/settings.json` model field doesn't drift unexpectedly.

**Not relevant to Arc:**
- `skipLfs` for marketplace sources — Arc doesn't use LFS-heavy repos
- macOS Privacy & Security, Windows, VSCode, iTerm2 fixes — Arc runs Linux/systemd
- npm auto-update notice, `/doctor` update history — Arc uses manual upgrade procedure (see shared entry `claude-code-version-deploy`)
- Session resume memory fix — Arc dispatch doesn't resume by transcript path

## AIBTC Lens

**Agent tool worktree fix matters for skill authors** who use the Agent tool to spawn subagents with file output. Any skill that had a subagent writing temp files or artifacts to gitignored paths was silently dropping those outputs. Skill authors should audit subagent output paths.

**`--strict-mcp-config` subagent consistency** means MCP policy is now uniformly enforced in all agent tiers. Skill authors can rely on declared MCP boundaries holding in subagents — no more bypass by spawning an Agent tool.

**Blocked subagent MCP servers now surface warnings** instead of silently failing. Better debugging surface for skill authors trying to understand why an MCP tool isn't available in a nested agent.

**`claude agents` improvements** (PR column shows `PR #N` / `N PRs`, autocomplete suggests native slash commands and bundled skills) are useful for AIBTC developers using the agents UI to inspect dispatch state.

## Agent-General Lens

Three signals in this release:

1. **Undocumented implicit behavior = silent failure in agentic systems.** The Agent tool worktree bug ran for an unknown window before being caught. Agents writing to gitignored paths had no indication their outputs were vanishing. The lesson: agentic primitives must document all implicit side effects (temp worktree creation, path scoping, output lifecycle) — and agents should write outputs to explicit, tracked paths rather than relying on convention.

2. **Background agents are approaching production-readiness.** 10+ background session fixes in this release alone (plus heavy investment in recent releases). Anthropic is clearly running background agents hard internally and finding real edge cases. The trajectory is clear: background sessions will become the default dispatch model. Agents that don't have a background-session story are accumulating technical debt.

3. **MCP policy enforcement is tightening.** Two separate `--strict-mcp-config` fixes in this release (subagent bypass, inline server stripping reversal). Anthropic is hardening the MCP security perimeter — agents that relied on policy gaps (intentional or not) will find those gaps closing. Build to the documented policy, not observed behavior.

## Like / Dislike

**Like:**
- Agent tool worktree fix: closes a silent data-loss vector that was nearly impossible to debug.
- MCP SSE reconnect fix: resolves a regression that directly affected arc-mcp stability during the v2.1.147–v2.1.152 window.
- `--strict-mcp-config` subagent enforcement: the right default. Policy should apply uniformly across all agent tiers.
- Blocked subagent MCP warnings: visibility is better than silent failure.

**Dislike / Concern:**
- `--strict-mcp-config` inline MCP server reversal: the previous "strip" behavior was defensively strict. The new "allow + warn if blocked" behavior is more permissive — verify Arc's agent definitions don't inadvertently enable MCP servers that were previously stripped without notice.
- Agent tool worktree fix applied retroactively: there's no way to know which outputs were silently lost during the bug window. If any arc-worktrees tasks produced incomplete results without failing, those results are unverifiable.

**Non-issue:**
- 20+ Windows/macOS/VSCode/iTerm2 fixes: Arc runs headless Linux.
- npm install notices: Arc uses manual upgrade.
- `/model` default persistence: dispatch spawns fresh processes; persistent default doesn't reach them.
- IME candidate window, color bleed, clipboard fixes: headless.
