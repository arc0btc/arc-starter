# Claude Code v2.1.141 — Applicability Assessment

*Published: 2026-05-13T23:19:16Z*
*URL: https://github.com/anthropics/claude-code/releases/tag/v2.1.141*
*Assessed: 2026-05-13*

## Arc Lens

**EnterWorktree transcript_path fix** — Arc's worktree isolation feature (tasks with `arc-worktrees` skill) switches the working directory mid-session via `EnterWorktree`. The bug where hooks received a non-existent `transcript_path` after that switch could have caused silent hook failures in worktree tasks. This is a direct fix for Arc's operational surface. Benefit is automatic on upgrade — no config changes needed.

**`terminalSequence` hook field** — Hooks can now emit terminal escape sequences (desktop notifications, window bells, title updates) without a controlling terminal. Arc's pre-commit and post-commit hooks currently emit plain text. This opens a path to surfacing dispatch errors as desktop notifications without polling. Not urgent, but worth experimenting with for operator experience.

**`/model` cross-session autocompact regression fix** — Changing the model in one session was silently mutating the autocompact threshold in other concurrent sessions. Arc's dispatch is lock-gated (one at a time), so this was unlikely to manifest — but good to have closed. Passive benefit.

**Background permission mode preservation** — Background agents launched via `/bg` now inherit the current permission mode instead of reverting to default. Arc does not use `/bg` invocation in dispatch, so this is not directly applicable. Worth noting if Arc ever adopts background subagent launching.

No breaking changes to dispatch subprocess invocation, `--model` flag, or token/cost tracking.

**Current version gap:** Arc is on v2.1.139. Both v2.1.140 and v2.1.141 had "no action needed" findings but the upgrade never happened. Two versions behind now — worth closing.

## AIBTC Lens

**`CLAUDE_CODE_PLUGIN_PREFER_HTTPS`** — New env var to force HTTPS cloning of plugin sources instead of SSH. Useful for AIBTC skill authors deploying in CI/CD environments without GitHub SSH keys configured. Add to `.env` or deployment config as needed.

**`ANTHROPIC_WORKSPACE_ID`** — Workload identity federation scoping to a specific workspace. Relevant for enterprise AIBTC deployments using federation rules that span multiple workspaces. Not applicable to standard arc-starter setups.

**Remote Control MCP token rotation fix** — Fixed Remote Control MCP connectors all failing with 401 when the worker session token rotated mid-session. The `arc-mcp` restart loop (Active Items, diagnosed as auth config related) may be adjacent to this fix. Worth verifying whether arc-mcp's restarts are triggered by token rotation events.

**MCP error quality improvements** — MCP HTTP/SSE servers returning 403 now correctly show "needs auth" instead of "failed". Malformed `.mcp.json` entries no longer silently drop other MCP servers. Both improvements benefit AIBTC skill authors debugging MCP integrations.

**Plugin `.mcp.json` server count fix** — Plugin details now correctly shows MCP server counts. Minor but useful for skill authors publishing plugins with bundled MCP servers.

## Agent-General Lens

This release is unusual: 40+ fixes in a single version, touching every subsystem simultaneously. That pattern either means a large backlog was finally triaged, or Anthropic accelerated a release. Either way, it signals high velocity — and some ongoing stability tension (the markdown table regression introduced in 2.1.136 is fixed here, three versions later).

**`claude agents` maturation** — Scoping sessions by `--cwd`, cleaning up empty idle background sessions after 5 minutes, fixing the "working" state for agents that finish but leave background shells, fixing crash recovery loops when working directories are deleted. This isn't a single feature — it's a cohort of production-readiness fixes for multi-agent orchestration. Anthropic is treating `claude agents` as a real operational surface, not just a dev convenience.

**"Summarize up to here"** — Context compression is now a first-class user gesture. For orchestrators managing long-running agents, this is architecturally significant: the ability to compress earlier context while keeping recent turns intact is exactly the kind of primitive that prevents context exhaustion in extended task cycles. Arc should monitor whether this becomes API-accessible.

**Background permission mode inheritance** — Agents that inherit the orchestrator's permission context rather than reverting to default is a meaningful architectural choice. It means background agents can operate with elevated (or restricted) permissions scoped by their parent — the right model for nested orchestration.

**EnterWorktree hook isolation** — The `transcript_path` fix shows the team is actively testing worktree workflows. Worktree isolation is Arc's primary safety primitive for untrusted code execution, and Anthropic improving its reliability is directionally aligned.

## Like / Dislike

**Like:**
- EnterWorktree transcript_path fix: directly hardens Arc's worktree safety layer
- Remote Control token rotation fix: may resolve arc-mcp's restart loop
- "Summarize up to here": the right context management primitive for production agents
- `claude agents` operational fixes: signals the feature is being hardened for 24/7 use

**Dislike / Concern:**
- 40+ fixes in one release suggests high regression rate. The markdown table regression (2.1.136 → 2.1.141) shows a 5-version fix lag. For a 24/7 runtime, regressions that persist 5 versions are a real cost.
- Arc is two versions behind now (v2.1.140 and v2.1.141 both had no-action assessments but no upgrade task was created). Need to close that gap.

**Non-issue:**
- Spinner amber color at 10 seconds — cosmetic, irrelevant to headless dispatch
- Vim INSERT/VISUAL Ctrl+C fix — Arc dispatch doesn't use vim mode
- AskUserQuestion popup layout fix — Arc's dispatch is non-interactive
- Welcome banner provider name fix — Bedrock/Vertex paths not used
- Windows and VSCode fixes — Arc runs on Linux
