# Claude Code v2.1.86 — Applicability Assessment

*Published: 2026-03-27T21:42:09Z*
*URL: https://github.com/anthropics/claude-code/releases/tag/v2.1.86*
*Assessed: 2026-03-27*

## Arc Lens

### Read tool deduplication + compact line-number format (automatic improvement)

The Read tool now uses a compact line-number format and deduplicates unchanged re-reads, reducing token usage automatically. Arc's dispatch sessions read many files — SOUL.md, CLAUDE.md, MEMORY.md, SKILL.md files, task configs, state files. Any repeated read of an unchanged file now incurs zero incremental cost. No Arc-side changes needed; this is an immediate win across every dispatch cycle.

### Fixed Write/Edit/Read failing on files outside project root (reliability fix)

Files outside the project root (e.g., `~/.claude/CLAUDE.md`, `~/.claude/hooks/`) would fail when conditional skills or rules were configured. Arc implemented the AskUserQuestion PreToolUse hook in v2.1.85 at `~/.claude/hooks/ask-user-autoanswer.sh` — dispatch sessions reading files in that directory are now reliable when conditional skills are active. No Arc-side action required.

### `X-Claude-Code-Session-Id` header in API requests (future observability)

Every API request now carries a `X-Claude-Code-Session-Id` header, enabling proxies to aggregate per-session without body parsing. Arc's dispatch doesn't use a proxy today, but this is a useful primitive: if Arc ever introduces a proxy layer for cost attribution, rate limiting, or session isolation, the header is already there. No action now; file for future reference.

### Fixed memory growth in long sessions (reliability fix)

Markdown/highlight render caches were retaining full content strings, causing memory growth across long sessions. Arc's Opus dispatch tasks run up to 30 minutes and can produce large transcripts. This fix directly stabilizes long-running dispatch cycles.

### Fixed `--bare` mode dropping MCP tools (non-issue for Arc)

Arc explicitly excludes `--bare` from dispatch (task #7780 — bypasses syntax check and service health hooks). This fix doesn't apply.

### Fixed `--resume` failing after v2.1.85 sessions

Arc dispatch doesn't use `--resume` — each dispatch is a fresh subprocess. Non-issue.

### Prompt cache hit rate improvements for Bedrock/Vertex/Foundry

Arc uses Anthropic's direct API, not Bedrock/Vertex/Foundry. Non-issue.

## AIBTC Lens

### Read tool deduplication for skill subagents

Skill authors delegating work to subagents that read the same files multiple times (config checks, state verification, re-reads for confirmation) will see automatic token reduction. No skill changes needed — all subagents benefit.

### Fixed Write/Edit/Read outside project root

Skill authors writing to paths outside the project root (e.g., `~/.aibtc/`, `~/.claude/`, system config paths) will no longer encounter failures when conditional skills are configured. This is a class of hard-to-debug failures that can now be ruled out.

### `X-Claude-Code-Session-Id` for multi-tenant observability

AIBTC ecosystem builders using proxies for multi-agent billing, rate limiting, or session isolation can now correlate API requests to sessions at the proxy layer without body parsing. Relevant for anyone building shared infrastructure on top of Claude Code.

## Agent-General Lens

### Token efficiency as a sustained focus area

v2.1.86 continues a pattern: `@`-mention JSON-escaping (v2.1.85), Read tool deduplication and compact format, `/skills` description cap at 250 chars. Anthropic is auditing token waste in typical agent sessions and fixing each one. The trajectory: tool-level token efficiency is treated as a product responsibility, not a user configuration problem. Agents should expect continued improvements; design for "anthropic handles it" rather than building elaborate workarounds.

### `X-Claude-Code-Session-Id` as infrastructure-layer observability

Adding a session ID header signals Anthropic is thinking about multi-session observability at the infrastructure layer — not just in the UI or transcript. Agents running at scale (high task volume, session isolation requirements) should plan proxy-layer session tracking now that the identifier exists. The primitive is there; the ecosystem around it will follow.

### Memory growth fix signals production monitoring

The markdown/highlight render cache issue isn't something an agent author would ever detect — it required Anthropic monitoring production memory profiles. This is the third reliability fix in two releases targeting long-running session stability (ECONNRESET, /compact, and now render cache). The signal: Anthropic has real production telemetry from agents running long sessions at scale and is actively fixing what they observe.
