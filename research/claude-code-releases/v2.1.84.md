# Claude Code v2.1.84 — Applicability Assessment

*Published: 2026-03-26T00:31:05Z*
*URL: https://github.com/anthropics/claude-code/releases/tag/v2.1.84*
*Assessed: 2026-03-26*

## Arc Lens

### CLAUDE_STREAM_IDLE_TIMEOUT_MS (action needed)

New env var configures the streaming idle watchdog threshold (default: 90s). Arc's dispatch cycles run up to 30 minutes and invoke Claude Code as a subprocess. Complex tasks — particularly Opus dispatch or tasks with long tool calls — can have extended idle periods between tool results. The 90s default may prematurely terminate these. Arc should evaluate setting this higher (e.g., 300,000ms = 5min) in the dispatch invocation environment, especially for Opus tasks.

### TaskCreated Hook (noteworthy)

A new `TaskCreated` hook fires when a task is created via the `TaskCreate` tool. Arc already has a hook framework (`db/hook-state/`). This hook could be used to monitor or log task creation in subagent workflows — relevant for dispatch sessions that delegate to Claude Code subprocesses. Low urgency but worth tracking as Arc's hook system matures.

### WorktreeCreate HTTP Hook (noteworthy)

`WorktreeCreate` hook now supports `type: "http"` — the hook can return the created worktree path via `hookSpecificOutput.worktreePath` in the response JSON. Arc's `arc-worktrees` skill isolates task execution in git worktrees. This allows HTTP-based event fanout on worktree creation. Not a blocking change — existing worktree behavior is unchanged.

### Global System-Prompt Caching + ToolSearch (good fix)

Global system-prompt caching now works when `ToolSearch` is enabled, including for users with MCP tools configured. Arc dispatches with `--dangerously-skip-permissions` and ToolSearch active in most sessions. This fix means Arc's large SOUL.md + CLAUDE.md + MEMORY.md context will now be cached properly in those sessions, reducing prompt costs on multi-tool dispatch cycles. Expect improved prompt cache rates going forward.

### Fixed: Subagent API 400 with --json-schema (bug fix)

Fixed workflow subagents failing with API 400 when the outer session uses `--json-schema` and the subagent also specifies a schema. Arc's dispatch runs Claude Code subprocesses. If Arc ever adopts structured output (`--json-schema`), this fix prevents a class of cascading failures in subagent chains. Not currently used by Arc, but good to know the failure mode is patched.

### Fixed: Cold-Start Race on Edit/Write (bug fix)

Fixed a cold-start race where core tools could be deferred without their bypass active, causing Edit/Write to fail with `InputValidationError` on typed parameters. Arc's dispatch opens fresh Claude Code sessions per task. This race was a real failure mode in short, fast-starting tasks. The fix reduces a class of intermittent dispatch failures — consistent with the "true failure rate ~10%" goal.

### MCP Changes

- **Tool descriptions capped at 2KB**: Arc uses `aibtc-mcp-server`. If any tool descriptions exceed 2KB, they will be silently truncated. No action needed from Arc's dispatch configuration, but the MCP server team should audit descriptions (see follow-up tasks).
- **MCP deduplication**: Local config wins over claude.ai connector. Consistent with Arc's approach of using locally-configured MCP.

### Startup Improvements

~30ms faster interactive startup; MCP REPL no longer blocks until all servers connect. Each dispatch cycle spawns a fresh Claude Code session — cumulative startup savings are meaningful at scale (199 tasks/day × 30ms = ~6 seconds/day, negligible but directionally right).

### paths: Frontmatter Accepts YAML List

Rules and skills `paths:` frontmatter now accepts a YAML list of globs (not just a single string). Arc's SKILL.md files don't use `paths:` frontmatter (they're loaded explicitly via task `skills` array). No direct impact, but skill authors should know this is now available.

## AIBTC Lens

### aibtc-mcp-server: 2KB Tool Description Cap

MCP tool descriptions and server instructions are now capped at 2KB. `aibtc-mcp-server` v1.43.0 has tools like `news_claim_beat`, `news_file_signal`, `send_inbox_message`, and identity/credential tools. Any tool description exceeding 2KB will be truncated silently, potentially breaking tool discovery or providing incomplete instructions to agents. The aibtcdev/skills team should audit current tool descriptions and ensure they stay under the limit. Particular risk: OpenAPI-derived descriptions or tools with inline JSON schema examples.

### skills paths: Frontmatter

Skill authors building arc-starter-compatible skills can now specify multiple `paths:` globs in frontmatter as a YAML list. Previously limited to a single glob. Relevant for skills that apply to overlapping but distinct directory structures.

### TaskCreated + WorktreeCreate Hooks

Skill authors building orchestration workflows can now react to task creation events (`TaskCreated`) and worktree creation over HTTP (`WorktreeCreate`). Useful for skills that need to observe or modify the task lifecycle at creation time.

### Subagent --json-schema Fix

Skills or workflows that use structured output in nested Claude Code sessions no longer risk the API 400 failure mode. Safe to adopt `--json-schema` in subagent chains now.

## Agent-General Lens

### Hook System Is Expanding

Three new/extended hooks in recent releases: `TaskCreated`, `WorktreeCreate` (http), and presumably more coming. Anthropic is building toward a richer event-driven architecture for Claude Code. Agents that want deep observability should invest in hook infrastructure now, before the surface area grows further. The direction is clear: hooks are becoming the primary extension point.

### Idle Timeout Configurability = Headless Agent Signal

`CLAUDE_STREAM_IDLE_TIMEOUT_MS` existing at all signals that Anthropic is aware agents run Claude Code in contexts where the 90s default is too conservative. This is an acknowledgment of the headless/autonomous use case. Agents with long-running tool chains (web scraping, code compilation, network waits) should audit their timeout configurations.

### MCP Description Discipline Will Be Enforced

The 2KB cap on MCP tool descriptions is a forcing function. OpenAPI-generated MCP servers that dumped full specs into tool descriptions will now be truncated. This is good for context hygiene but will break existing integrations that relied on verbose descriptions. The broader signal: Anthropic is going to keep constraining context bloat from MCP. Agent builders should write minimal, precise tool descriptions — not inline documentation.

### Prompt Cache Reliability Improvements

The ToolSearch + global caching fix, combined with improved p90 cache rates, suggests Anthropic is actively investing in cache infrastructure. For agents paying at API rates, this is meaningful cost reduction. Arc should expect the `api_cost_usd` tracking in cycle_log to show improved efficiency.

### Startup Performance as a First-Class Concern

Multiple startup improvements (30ms REPL speedup, MCP non-blocking REPL render, partial-clone mass-download fix) signal Anthropic treating startup latency as a real product metric. For agents spawning many Claude Code subprocesses per day, this compounds meaningfully. Anthropic is moving toward treating Claude Code as a proper daemon/service, not just an interactive terminal tool.

## Like / Dislike

**Like:**
- `CLAUDE_STREAM_IDLE_TIMEOUT_MS` — finally. Long-running dispatch tasks have been at risk from a conservative watchdog. Now configurable.
- `TaskCreated` hook — expands Arc's event-driven hook framework; more observability surface
- Global system-prompt caching now works with ToolSearch — this was likely costing Arc real money every cycle. Good fix.
- Fixed cold-start race on Edit/Write — reduces intermittent dispatch failures; this one was real
- `WorktreeCreate` http hook — opens door for external worktree event handling

**Dislike / Concern:**
- MCP 2KB description cap is silent truncation, not an error — truncated descriptions will make tools appear partially broken without obvious diagnostics. Should be a warning, not a silent cap.
- No mention of whether `CLAUDE_STREAM_IDLE_TIMEOUT_MS` applies to non-interactive subprocess invocations or only interactive sessions. Need to test.
- The idle-return /clear nudge at 75 minutes could interfere with very long interactive sessions used for development; doesn't affect dispatch but is mildly annoying for humans.

**Non-issue:**
- PowerShell tool — Arc runs on Linux
- Deep links (`claude-cli://`) — not used in headless dispatch
- Voice push-to-talk fixes — irrelevant for Arc
- Mouse/keyboard/IME fixes — Arc is headless
- VSCode rate limit banner — Arc doesn't use VS Code extension
- `#123` bare reference auto-link removal — doesn't affect dispatch output
