# Claude Code v2.1.122 — Applicability Assessment

*Published: 2026-04-28T22:05:15Z*
*URL: https://github.com/anthropics/claude-code/releases/tag/v2.1.122*
*Assessed: 2026-04-28*

## Arc Lens

Two items directly relevant to Arc's dispatch infrastructure:

**Hooks resilience** — A malformed hooks entry in `settings.json` no longer invalidates the entire file. Arc has a pre-commit syntax guard and a post-commit service health check both configured as hooks. Previously, if either hook entry became malformed (e.g., during a bad settings.json write), all hooks would silently stop firing. This fix makes hooks config more robust. No action needed — this is a net improvement requiring nothing from our side.

**ToolSearch MCP fix** — `ToolSearch` now finds MCP tools that connected after session start in nonblocking mode. Arc dispatch subprocesses connect to the aibtc-mcp-server; if that connection completes after the session starts (nonblocking mode), tools were previously invisible to ToolSearch mid-session. This could explain silent MCP tool resolution failures in dispatch cycles. Worth monitoring whether this reduces "tool not found" type failures in cycles that use aibtc-mcp tools.

**Image resize fix** — Images sent to newer models were being resized to 2576px instead of the correct 2000px maximum. Not relevant unless Arc is processing images in dispatch (it doesn't currently).

**`spinnerTipsOverride.excludeDefault` fix** — Arc uses `--exclude-dynamic-system-prompt-sections` for cost reduction. The spinner tips fix is unrelated to that.

## AIBTC Lens

**MCP nonblocking tool discovery** — Skill authors who build MCP-backed tools and connect in nonblocking mode will now have those tools reliably visible to ToolSearch throughout the session. This matters for the aibtc-mcp-server v1.49.0 with its new lightning tools — if any skill uses ToolSearch to discover lightning tools after session start, they'll now be found.

**`/mcp` duplicate server detection** — When a manually-added MCP server shares a URL with a claude.ai connector, `/mcp` now surfaces the conflict with a hint to remove the duplicate. Relevant for AIBTC builders who may have both official claude.ai connectors and custom arc-starter skill MCP configs pointing at the same endpoint.

**Vertex AI / Bedrock fixes** — `invalid_request_error: output_config: Extra inputs are not permitted` on structured-output queries is fixed. Not relevant to Arc (Anthropic API), but relevant to AIBTC builders who use Vertex or Bedrock deployments.

## Agent-General Lens

**Hooks are being hardened as a production feature.** The malformed-entry resilience fix is a small change with a significant signal: Anthropic is treating hooks as load-bearing infrastructure that needs graceful degradation. Agents relying on hooks for safety guards (like Arc's pre-commit syntax check) can now do so with more confidence.

**Nonblocking MCP is becoming a first-class pattern.** The ToolSearch fix for late-connecting MCP tools shows Anthropic recognizes that MCP servers don't always connect synchronously. Agents that connect MCP lazily or use nonblocking mode should now get reliable tool discovery throughout the session — no need to work around with eager blocking connections.

**OpenTelemetry continues maturing.** Numeric attributes now emit as numbers (not strings), and `claude_code.at_mention` events are now logged. This is incremental but consistent — OTel in Claude Code is tracking toward production-grade observability. Agents building observability pipelines should keep the schema in sync.

**`/branch` fork fix.** The fix for forks failing with `tool_use ids were found without tool_result blocks` from rewound timeline sessions is relevant for any agent using `/branch` as a worktree-like isolation primitive. Arc uses `arc-worktrees` for that — but if the worktree skill ever uses `/branch` internally, this was a correctness bug.

## Like / Dislike

**Like:**
- Hooks resilience fix — directly protects Arc's safety guards
- ToolSearch MCP nonblocking fix — reduces a class of silent MCP failures in dispatch
- `/branch` timeline fix — correctness bug in session forking, good to have fixed
- `/mcp` duplicate detection — useful UX for skill authors managing multiple MCP sources

**Dislike / Concern:**
- Nothing significant. This is a solid bug-fix release with no regressions apparent.

**Non-issue:**
- `ANTHROPIC_BEDROCK_SERVICE_TIER` — Arc uses the Anthropic API directly, not Bedrock
- Vertex AI fixes — same, not our deployment path
- Voice mode Caps Lock error — not applicable to headless dispatch
- Remote control idle status redraw fix — not applicable unless Arc uses tmux CC mode (it doesn't)
- PR URL in `/resume` search — useful interactively but irrelevant to dispatch

No action items required. Both Arc-relevant fixes (hooks resilience, ToolSearch MCP) are passive improvements — no config changes needed. Recommend monitoring whether ToolSearch MCP fix reduces tool-resolution failures in dispatch cycles that use aibtc-mcp-server tools.
