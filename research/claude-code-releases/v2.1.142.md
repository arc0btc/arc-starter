# Claude Code v2.1.142 — Applicability Assessment

*Published: 2026-05-14T22:55:10Z*
*URL: https://github.com/anthropics/claude-code/releases/tag/v2.1.142*
*Assessed: 2026-05-14*

## Arc Lens

**MCP_TOOL_TIMEOUT fix (actionable):** Remote HTTP and SSE MCP servers were previously capped at 60s regardless of the configured timeout — `MCP_TOOL_TIMEOUT` was parsed but not applied to per-request fetch timeouts. This is now fixed. Arc uses arc-mcp; if it connects via HTTP/SSE transport, tool calls could have been silently timing out at 60s. Worth auditing whether arc-mcp is configured with an explicit `MCP_TOOL_TIMEOUT` and whether 60s was causing hidden failures.

**Worktree fix (likely N/A):** Fixed background sessions not recognizing pre-existing git worktrees — previously blocked Edit calls while EnterWorktree refused to create a duplicate. Arc's worktree isolation (`arc-worktrees` skill) uses `--print` mode dispatch, not the background sessions daemon. So this fix doesn't directly apply. No action needed, but worth monitoring if worktree tasks start using the `claude agents` daemon path in the future.

**Reactive compaction improvement (passive benefit):** "The first summarize attempt now seeds from the original request's overflow size, avoiding a wasted near-full-context retry." Arc hits context ceilings on long dispatch cycles (patterns.md consolidation, omnibus signal research). This improvement means fewer wasted retry cycles at overflow. No config change needed — automatic improvement.

**`claude agents` flag expansion (directional signal for Arc):** New flags `--model`, `--effort`, `--permission-mode`, `--dangerously-skip-permissions`, `--settings`, `--mcp-config`, `--add-dir`, `--plugin-dir` are now exposed on `claude agents`. Arc's current `--print` dispatch already uses `--model`, `--effort`, and `--permission-mode` equivalents. This closes the feature gap between `--print` and background-session dispatch. If Arc ever moves to daemon-based dispatch, the API parity is now there.

**Hook config error improvement (verify):** Prompt- and agent-type hooks for `SessionStart`/`Setup`/`SubagentStart` now emit a clear error ("use a command-type hook instead"). Arc uses `session-start.sh` as a command-type hook. Verify the hook type in `.claude/settings.json` is set to `command` — if it's misconfigured as `prompt` or `agent`, it would have silently misbehaved before; now it errors visibly.

**Fast mode → Opus 4.7 (not applicable):** Arc doesn't use fast mode interactively. Arc's dispatch uses explicit `--model` and `--effort` flags. No change needed.

**Daemon crash-loop fix after upgrade (passive benefit):** Dispatch upgrades Claude Code periodically via `claude-code-version-deploy`. The fix for the daemon not exiting cleanly after binary upgrade prevents crash-loops on the deleted path. Arc restarts services post-upgrade; this reduces the chance of orphaned daemon processes after upgrade.

## AIBTC Lens

**Plugin SKILL.md surfacing:** Plugins with a root-level `SKILL.md` and no `skills/` subdirectory are now surfaced as a skill in Claude Code's UI. Arc skills use `SKILL.md` extensively, but these are Arc's own skill system — not Claude Code plugins. Skill authors building Claude Code plugins that follow Arc's SKILL.md convention will now get automatic skill surfacing in the `/plugin` pane. Low impact for current arc-starter skills, higher relevance for any arc-starter-compatible plugins published to the Claude Code plugin registry.

**MCP timeout fix:** AIBTC ecosystem services that Arc connects to via MCP (e.g., arc-mcp, Hiro API proxies, any SSE-based integrations) were silently capped at 60s. This fix is a reliability improvement for integrations that have latent tool calls (Pyth VAA lookups, sBTC operations).

## Agent-General Lens

**`claude agents` is becoming the primary dispatch API.** This release brings `--model`, `--effort`, `--permission-mode`, and `--dangerously-skip-permissions` to `claude agents`. Anthropic is converging background-session management with the flags that `--print` mode already supports. Direction: `claude agents` + daemon is the emerging first-class agent runtime, not `--print` + subprocess. Arc should monitor whether the `claude agents` flow gains capabilities that `--print` mode loses or doesn't get.

**Daemon reliability hardening.** Multiple fixes for crash-loops (binary upgrade, macOS sleep/wake, Chrome extension, Windows network drives) signal that the daemon-based model is being hardened for production use. This is consistent with Anthropic investing in `claude agents` as the persistent-agent primitive.

**Plugin ecosystem maturing.** The `/plugin` browse pane fixes (0 installs display, advisory naming, cache cleanup) suggest the plugin registry is being actively developed. The SKILL.md surfacing change is a deliberate bridge between the plugin format and the skill format — convergence between how Arc thinks about skills and how Claude Code thinks about plugins.

## Like / Dislike

**Like:**
- MCP_TOOL_TIMEOUT fix is a real reliability improvement — silent 60s caps on tool calls were a hidden failure mode
- Reactive compaction improvement is directly useful for Arc's long dispatch cycles
- `claude agents` flag parity with `--print` mode is welcome — less divergence to track
- Daemon upgrade crash-loop fix makes `claude-code-version-deploy` safer

**Dislike / Concern:**
- No explicit changelog entry for any `--print` mode improvements — Anthropic's investment is clearly in the daemon/agents flow. Arc should watch whether `--print` mode becomes second-class over the next few releases.
- The macOS sleep/wake daemon fix is macOS-only; Arc runs on Linux (systemd). Daemon clock-jump detection may or may not apply to Linux suspend states.

**Non-issue:**
- Fast mode → Opus 4.7 default: Arc doesn't use fast mode interactively
- Session title URL derivation fix: N/A for Arc's non-interactive dispatch
- Terminal color fixes (Apple Terminal 256-color): N/A for Arc's headless operation
- `/web-setup` GitHub App warning: N/A
