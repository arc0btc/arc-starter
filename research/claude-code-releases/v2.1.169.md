# Claude Code v2.1.169 — Applicability Assessment

*Published: 2026-06-08T21:57:18Z*
*URL: https://github.com/anthropics/claude-code/releases/tag/v2.1.169*
*Assessed: 2026-06-08*

## Arc Lens

**`--safe-mode` / `CLAUDE_CODE_SAFE_MODE`**: Disables all customizations (CLAUDE.md, plugins, skills, hooks, MCP servers) for troubleshooting. Arc dispatches Claude Code subprocesses with `CLAUDE.md` + skills intentionally loaded — safe mode is explicitly for debugging, not standard dispatch. No config change needed. Useful diagnostically if a dispatch cycle misbehaves and we need to isolate whether Arc's own CLAUDE.md/skills are the cause. Could add a `--safe` flag to `arc run` for debug sessions.

**`disableBundledSkills` / `CLAUDE_CODE_DISABLE_BUNDLED_SKILLS`**: Hides built-in slash commands, bundled skills, and workflows from the model. Arc dispatches headless (`-p` flag), so slash commands and bundled skills don't surface anyway — this is a non-issue for dispatch. Potentially useful if bundled skills ever pollute the model's tool context in unexpected ways.

**`CLAUDE_CODE_SAFE_MODE` env var**: Can be set in dispatch subprocess env block (dispatch.ts). No reason to set it by default, but document it for troubleshooting runbooks.

**Background agents fix — `ANTHROPIC_MODEL` env values now respected**: Fixed background sessions ignoring project-level `env` values like `ANTHROPIC_MODEL`. Arc's dispatch.ts sets env vars for the subprocess — this fix ensures those are reliably applied in background/worker contexts. **Relevant if Arc ever uses background sessions for parallel dispatch.** For current sequential dispatch, impact is low.

**`claude -p` hang fix on Windows (regression from 2.1.161)**: Arc runs on Linux — not applicable.

**Vertex/Foundry idle timeout restored (5min default)**: Arc uses `ANTHROPIC_API_KEY` path, not Vertex/Foundry. No config change needed.

**`TaskCreate` reliability improvements**: Malformed inputs auto-repaired; validation errors for unloaded tools include schema. Arc's dispatch context uses `TaskCreate` when creating follow-up tasks from within a session. Better validation messages reduce debugging friction.

**`/cd` command**: Moves session to a new working directory without breaking prompt cache. Useful for interactive sessions; no dispatch relevance.

**CLAUDE.md length warning now scales with model context window**: Arc's CLAUDE.md is long (~50k chars). This removes a spurious warning that might have appeared during dispatch when using models with larger context windows (opus-4-8 has 200k). No action needed, just noise reduction.

**Conclusion:** No breaking changes. The background-session `ANTHROPIC_MODEL` env fix is latent-relevant if Arc expands to parallel dispatch. The `--safe-mode` flag is a useful troubleshooting primitive to document.

## AIBTC Lens

**`disableBundledSkills`**: Skill authors building on arc-starter can now suppress built-in slash commands and workflows when shipping agents. Useful for production agents that should have a clean, deterministic tool surface (no `/help`, `/workflows` etc. leaking to users). AIBTC agent deployments might consider setting `CLAUDE_CODE_DISABLE_BUNDLED_SKILLS=1` to prevent user confusion.

**Enterprise MCP policy fixes**: `allowedMcpServers`/`deniedMcpServers` now enforced correctly on reconnect and first session after install. AIBTC's MCP server (`aibtc-mcp-server`) is used by ecosystem builders — this fix means enterprise managed policy enforcement is now reliable. Important for any AIBTC teams using managed Claude Code deployments.

**`claude agents --json` fix — blocked/dispatched sessions now included, `--all` adds completed**: Agents building multi-agent systems that query `claude agents --json` to track live subagent state will now get complete data. `id` and `state` fields added. Arc-workflows or any skill managing concurrent background sessions benefits here.

**Background agents preserve flags across retire→wake**: `--ide`, `--chrome`, `--bare`, `--remote-control` preserved across worker restarts. Ecosystem builders building persistent agents on the background session pattern get better reliability.

**Conclusion:** The `disableBundledSkills` setting is the most actionable for AIBTC agent deployments — recommend documenting in arc-starter CLAUDE.md as a production deployment pattern.

## Agent-General Lens

**`--safe-mode` signals a maturing troubleshooting story.** Anthropic is acknowledging that customizations (hooks, MCP, CLAUDE.md) create emergent complexity that needs a clean isolation path. This is a sign that the customization surface area is large enough to warrant a "factory reset" mode — maturity indicator, not a concern.

**`disableBundledSkills` = controlled tool surface.** The ability to strip out Anthropic's bundled slash commands for production deployments shows movement toward agent-first usage patterns where the operator (not the user) controls what the model sees. This is the right direction for multi-agent and headless deployments.

**Background session reliability hardening (multiple fixes this release)**: flags preserved across retire→wake, shared-checkout edits blocked in non-worktree context, `claude agents` data completeness. This release has significant investment in multi-agent / background session correctness. Arc's current architecture is single-dispatch; Anthropic is investing heavily in background/parallel agents. Watch this space — the gap between Arc's sequential model and CC's parallel model is growing.

**MCP enterprise policy fixes**: Managed deployments are a real product concern for Anthropic. Enterprise MCP policy enforcement being broken across reconnects/first-sessions is a trust issue. Fix is good; the fact it was broken this long suggests enterprise deployment patterns are relatively new to this surface.

**Conclusion:** This release continues Anthropic's push to make Claude Code viable for production multi-agent deployments. The troubleshooting (safe-mode) and control (disableBundledSkills) primitives suggest increasing operator-side control. Arc should track background session improvements — sequential dispatch may become a bottleneck relative to native CC parallel agent support.

## Like / Dislike

**Like:**
- `--safe-mode` is a proper diagnostic primitive. When a dispatch misbehaves, we now have a clean isolation tool rather than having to temporarily comment out config.
- `disableBundledSkills` is clean for production deployments — suppressing built-in commands from the model surface is something Arc has needed conceptually.
- CLAUDE.md length warning scaled to context window — stops spurious warnings for opus-class models. A small but sensible improvement.
- Multiple background session reliability fixes in one release — shows sustained investment in multi-agent correctness.

**Dislike / Concern:**
- `--safe-mode` disables EVERYTHING including hooks and MCP. There's no partial safe mode (e.g., "disable MCP but keep hooks"). The binary nature limits its diagnostic usefulness for isolating specific subsystems.
- The `ANTHROPIC_MODEL` env fix (background sessions ignoring project env) was a silent correctness bug. The fact it existed and wasn't caught earlier suggests background session testing coverage is thin. Watch for similar environment-propagation bugs as Arc considers parallel dispatch.

**Non-issue:**
- Windows-specific fixes (git credential popup, `claude -p` hang, WSL agents view, MCPB cache, PID lock files, macOS 30-50ms stall) — Arc runs on Linux.
- Remote Control / OAuth refresh fixes — Arc uses API key path, not claude.ai credentials.
- Vertex/Foundry idle timeout — Arc doesn't use Vertex/Foundry.
- UI polish (footer hints, color contrast, cursor navigation in long inputs) — dispatch is headless.
