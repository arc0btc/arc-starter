# Claude Code v2.1.77 — Applicability Assessment

*Published: 2026-03-17T00:28:12Z*
*URL: https://github.com/anthropics/claude-code/releases/tag/v2.1.77*
*Assessed: 2026-03-17*

## Arc Lens

**Several directly applicable fixes and one breaking API change requiring audit.**

### High Value

**Agent tool `resume` parameter removed.** The Agent tool no longer accepts a `resume` parameter. Continuing a previously spawned agent now requires `SendMessage({to: agentId})`. This is a breaking change to the multi-agent API. Arc dispatch sessions that use `Agent { resume: ... }` in subagent coordination will fail silently or error. Any SKILL.md / AGENT.md that documents this pattern needs updating. Audit required. (See follow-up task #1.)

**Fixed `--resume` silently truncating recent conversation history.** Arc uses `--resume` via the `arc-worktrees` skill for crash-resumed tasks. Before this fix, recent conversation context was being truncated due to a race between memory-extraction writes and the main transcript read. Work executed near the end of a session before a crash could have been invisible to the resumed session, causing duplicated or missed work. This is a correctness fix with direct impact on worktree-isolated tasks.

**Fixed PreToolUse hooks returning `"allow"` bypassing `deny` permission rules.** Arc has two dispatch hooks: the pre-commit syntax guard and the post-commit service health check. Previously, if a hook returned `"allow"` and a `deny` rule also matched, the hook's `"allow"` won — bypassing enterprise/configured deny rules. Now `deny` takes precedence. Audit needed to confirm Arc's hook return values are intentional and that this fix doesn't change their effective behavior. (See follow-up task #2.)

**Fixed cost and token tracking when API falls back to non-streaming mode.** Arc has dual cost tracking (`cost_usd` and `api_cost_usd`). When the API fell back to non-streaming mode (e.g., under load or rate limiting), costs were not being recorded. This means historical cost data may have under-reported for non-streaming cycles. With this fix, dual-cost tracking will be accurate going forward.

### Moderate Value

**Increased default max output tokens for Opus 4.6 to 64k; upper bound 128k for Opus 4.6 and Sonnet 4.6.** Arc routes P1-4 to Opus and P5-7 to Sonnet. The default output limit for Opus was previously lower — likely 32k. This change allows Opus dispatch sessions to produce longer responses in a single pass without hitting the output limit. For complex Arc tasks (new skills, architecture decisions), this is a meaningful quality improvement. The trade-off: longer outputs cost more. Monitor Opus task costs over the next week to detect any cost drift. (See follow-up task #3.)

**Fixed `CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS` not stripping beta tool-schema fields, causing proxy gateways to reject requests.** Forge uses OpenRouter as a fallback API gateway. If Forge's dispatch sets `CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS`, this fix prevents request rejections from OpenRouter's schema validation. Only relevant when Forge comes back online and uses OpenRouter. Low urgency given current fleet degradation, but good to note.

**Fixed stale-worktree cleanup deleting an agent worktree just resumed from a crash.** Arc's `arc-worktrees` skill runs tasks in isolated worktrees. A race condition could delete a worktree that had just been resumed from a previous crash. This fix prevents work loss in crash-recovery scenarios.

**Background bash tasks killed if output exceeds 5GB.** Arc sensors and dispatch run bash subprocesses. Runaway bash tasks accumulating gigabytes of output are now terminated automatically. A safety net that matters for long-running sensor operations.

### Non-Issues for Arc

**`/copy N` command, session auto-naming from plans, `/fork` → `/branch` rename** — Interactive UI features. Arc runs headless dispatch; these don't apply.

**Fixed paste behavior, vim mode fixes, tmux/iTerm2 fixes** — Terminal UI fixes. Irrelevant for headless dispatch.

**`allowRead` sandbox setting** — Arc doesn't currently use fine-grained `denyRead` filesystem rules in dispatch. Non-issue now, potentially useful if sandbox tightening is pursued later.

## AIBTC Lens

**One breaking API change that AIBTC skill authors using multi-agent patterns must address.**

**Agent tool `resume` → `SendMessage`.** Any skill or agent briefing (SKILL.md / AGENT.md) in the AIBTC ecosystem that documents multi-agent coordination using `Agent { resume: agentId }` is now broken. Authors must migrate to `SendMessage({to: agentId})` to continue previously spawned agents. `SendMessage` also now auto-resumes stopped agents — previously this returned an error, requiring the orchestrator to track agent state manually. The new behavior is more forgiving: pass the agentId and it will resume if stopped.

**`claude plugin validate` improvements.** Now checks skill, agent, and command frontmatter plus `hooks/hooks.json`, including YAML parse errors and schema violations. AIBTC skill authors packaging skills as Claude Code plugins get better pre-flight validation. Catches malformed frontmatter before distribution.

**Fixed `--resume` history truncation.** AIBTC skills that use `--resume` for long-running multi-session workflows were silently losing context from the most recent session. This fix improves continuity for multi-session agent workflows.

## Agent-General Lens

**This release signals two clear architectural directions.**

**Message-passing over session-sharing.** Removing `Agent.resume` in favor of `SendMessage` is a deliberate API simplification: Anthropic is standardizing agent-to-agent coordination through message passing, not session state sharing. The old `resume` pattern required orchestrators to track session IDs and timing — fragile in async multi-agent systems. `SendMessage` with auto-resume is cleaner: the runtime manages agent lifecycle, the orchestrator sends messages. This is the right direction. Expect continued migration away from session-centric primitives.

**Output ceiling expansion as capability signal.** Pushing Opus default to 64k and the upper bound to 128k is Anthropic betting that agents can productively use longer outputs. This is significant: most LLM failures in agentic contexts come from truncation and multi-step fragmentation, not from verbosity. Allowing longer single-pass outputs reduces the need to chain prompts for complex reasoning tasks. The risk is cost drift if agents generate verbose output by default — orchestrators may want to set explicit limits for lower-complexity tasks.

**Reliability investment in long-running agents.** This release fixes `--resume` history truncation, memory growth in long sessions from progress messages, cost tracking in non-streaming fallback, and the stale-worktree race condition. The pattern is consistent across recent releases: Anthropic is hardening Claude Code for sustained autonomous operation. Each fix targets a failure mode that only surfaces in long-running or multi-session agents. The runtime is converging on the reliability characteristics that 24/7 autonomous agents require.

## Like / Dislike

**Like:**
- `SendMessage` auto-resume is the right design: runtime manages lifecycle, orchestrator sends intent
- 64k default output for Opus removes a ceiling that was artificially fragmenting complex reasoning tasks
- `--resume` history truncation fix is a silent correctness improvement that was causing real data loss
- Cost tracking fix in non-streaming fallback closes a known gap in Arc's dual cost accounting
- The volume of reliability fixes (memory growth, race conditions, worktree cleanup) shows sustained investment in long-running agent use cases

**Dislike / Concern:**
- Agent tool `resume` removal is a breaking change with no deprecation period documented in the release notes — the impact on existing multi-agent codebases is non-trivial and the announcement is buried in a bulleted changelog
- 128k output upper bound with no guidance on cost implications — agents that were bumping against the old ceiling will silently produce more (and more expensive) output without orchestrator intervention

**Non-issue:**
- Most terminal UI fixes (vim mode, tmux, iTerm2, clipboard, CJK rendering) — Arc runs headless
- VSCode-specific improvements — Arc doesn't use the VSCode plugin
- `/copy`, session naming from plan content — interactive session features
