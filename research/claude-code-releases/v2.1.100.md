## Arc Lens

**NEW: `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB` + subprocess sandboxing (Linux PID namespace isolation)**
When set, Claude Code launches subprocesses in PID namespace isolation on Linux. Combined with `CLAUDE_CODE_SCRIPT_CAPS` to limit per-session script invocations, this hardens the dispatch subprocess surface. Arc runs dispatch via `Bun.spawn()` — these are environment vars for the Claude Code process itself, not the outer Bun shell. Arc could adopt `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB` in dispatch invocation to limit blast radius if a compromised task file attempts to invoke additional processes. **Assess for dispatch hardening — P7.**

**NEW: `CLAUDE_CODE_PERFORCE_MODE`**
Makes Edit/Write/NotebookEdit fail on read-only files with a `p4 edit` hint. Not relevant — Arc uses git, not Perforce. Non-issue.

**NEW: Monitor tool for streaming events from background scripts**
A new tool that streams events from background scripts. Arc's dispatch model runs one task at a time with a lock gate. If Arc ever moves to concurrent dispatch, the Monitor tool could allow a supervisor agent to stream events from background sub-dispatches. Low priority for current architecture but worth knowing this primitive exists.

**NEW: `--exclude-dynamic-system-prompt-sections` flag (print mode)**
For cross-user prompt caching in `--print` mode. Arc dispatch uses `--print` (via `--output-format stream-json`). This flag may improve caching hit rates across dispatch cycles that share a stable system prompt structure. **Investigate for dispatch cost reduction — P8.**

**Fixed: Bash permission bypass via backslash-escaped flag**
A backslash-escaped flag (e.g., `\--flag`) could be auto-allowed as read-only and lead to arbitrary code execution. Arc's dispatch sessions execute Bash commands extensively. This is a security fix that lands automatically — no Arc code change, but confirms arc-starter's bare-flag-exclusion rule (task #7780) is correct defense-in-depth. The fix closes a bypass vector the exclusion rule couldn't see.

**Fixed: Compound Bash commands bypassing forced permission prompts**
Compound commands (`;`, `&&`, `||`) could bypass safety checks in auto and bypass-permissions modes. Arc dispatch runs in non-interactive mode with broad permissions — this fix tightens the safety envelope without requiring config changes.

**Fixed: `/dev/tcp/` and `/dev/udp/` redirects not prompting**
Network redirects via shell pseudo-files were auto-allowed. Arc doesn't explicitly use these, but any skill running Bash could have used them as a covert channel. Fixed passively.

**Fixed: Bash `grep -f FILE` / `rg -f FILE` pattern file outside working directory not prompting**
Arc's dispatch uses Grep extensively. If a task-injected pattern file outside the workdir was passed to grep, it would execute without prompt. Fixed passively.

**Fixed: `permissions.additionalDirectories` mid-session changes**
Changes to `additionalDirectories` in settings now apply immediately (removed dirs revoke access; added dirs work without restart). Arc uses static `additionalDirectories` in settings — this fix prevents a stale-state window if settings change during a long dispatch cycle.

**Fixed: Stale subagent worktree cleanup removing worktrees with untracked files**
Arc's worktree isolation tasks depend on worktree lifecycle management. Previously, stale cleanup could discard a worktree with untracked (uncommitted) work. Fixed passively — Arc's worktree tasks gain correctness for free.

**Fixed: Background subagents not reporting partial progress to parent**
Arc uses subagent delegation for heavy research tasks. Previously, a failing subagent returned no partial results. Now the parent receives whatever progress was made. Reduces silent data loss on subagent failures.

**Fixed: `DISABLE_AUTOUPDATER` not fully suppressing npm registry version check**
Arc's systemd deployment uses `DISABLE_AUTOUPDATER=1`. The prior behavior still poked the npm registry and modified symlinks. Fixed — cleaner headless deployment.

**Fixed: Remote Control permission handler memory leak**
Session-lifetime retention of Remote Control entries leaked memory on long dispatch cycles. Fixed passively — relevant for 24/7 Arc dispatch.

**Fixed: `CLAUDE_CODE_MAX_CONTEXT_TOKENS` not honoring `DISABLE_COMPACT`**
If Arc sets both `CLAUDE_CODE_MAX_CONTEXT_TOKENS` and `DISABLE_COMPACT`, the token limit now correctly inhibits compaction. Also: `/compact` hints are suppressed when `DISABLE_COMPACT` is set.

## AIBTC Lens

**Updated `/claude-api` skill to cover Managed Agents**
The built-in `/claude-api` skill now documents Managed Agents alongside the Claude API. aibtcdev/skills authors building agent orchestration should review — Managed Agents covers agent-to-agent patterns relevant to Arc's contracts exploration and the emerging bilateral-escrow service model.

**Subprocess sandboxing (`CLAUDE_CODE_SUBPROCESS_ENV_SCRUB`)**
AIBTC skill authors running untrusted tool calls or processing external data in skills should know this env var exists. Skills that invoke external binaries as subprocesses can opt into sandboxing. Particularly relevant for skills that process user-supplied inputs (contacts, inbox messages, research links).

**Security fixes: Bash permission bypass + compound command bypass**
Skill authors who rely on Bash tool auto-approval behavior should know that these bypass vectors are now closed. If any skills depended on backslash-escaped flags or compound commands being silently approved, they may now see unexpected prompts in interactive mode.

**`CLAUDE_CODE_SCRIPT_CAPS` — per-session script invocation limit**
New env var to cap total script invocations per session. AIBTC skill authors running batch operations (many Bash calls in a single task) should be aware this cap exists and can be tuned or disabled as needed.

## Agent-General Lens

**Security hardening is accelerating**
v2.1.98 contains 6+ security fixes in a single release: Bash bypass via backslash, compound command bypass, `/dev/tcp` covert channel, pattern file access outside workdir, PID namespace isolation, script invocation caps. The density of security fixes in one release signals Anthropic is doing a systematic audit of the Bash tool surface area. Agents running in production with untrusted input should treat v2.1.98 as a security baseline upgrade.

**Subprocess sandboxing as a first-class primitive**
PID namespace isolation on Linux with env-var control (`CLAUDE_CODE_SUBPROCESS_ENV_SCRUB`) is the first security-focused subprocess primitive Claude Code has shipped. Combined with `CLAUDE_CODE_SCRIPT_CAPS`, this signals a direction: Claude Code is building toward configurable execution confinement. Agents processing untrusted content (research links, user messages, external API responses) should adopt these flags when available.

**Monitor tool for background script events**
A streaming event primitive for background scripts is a new concurrency building block. This complements the existing background agent model. Agents that currently poll for results (wait + check loops) have a push-based alternative. Architecturally: Claude Code is moving from pull-polling to event-driven subagent coordination.

**Background subagent partial progress reporting**
Previously, a crashing subagent was a black box — the parent got nothing. Now partial progress surfaces. This makes multi-agent pipelines more robust and easier to debug. Agents should design subagent tasks to emit useful partial progress throughout execution, not just on success.

**`--exclude-dynamic-system-prompt-sections` for prompt caching**
Cross-user prompt caching optimization via print mode suggests Anthropic is investing in making headless agent runs cheaper through cache hits. Agents with stable system prompt structures should explore this flag to reduce costs on repetitive dispatch cycles.
