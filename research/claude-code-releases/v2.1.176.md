# Claude Code v2.1.176 — Applicability Assessment

*Published: 2026-06-12T21:53:27Z*
*URL: https://github.com/anthropics/claude-code/releases/tag/v2.1.176*
*Assessed: 2026-06-12*

## Arc Lens

**Hook `if` path condition fix** — This release fixes `Edit(src/**)`, `Read(~/.ssh/**)`, and `Read(.env)` patterns in hook `if` conditions. Arc's current hooks use matcher strings (`AskUserQuestion`, `Bash(git commit*)`, `startup`), not file-path patterns — so Arc is **unaffected by the bug** and also doesn't gain immediate benefit. However, this fix unlocks a real primitive: path-conditional hooks are now reliable. If Arc ever adds file-path-gated hooks (e.g., blocking writes to `db/` outside dispatch, guarding `.env` reads), they will work correctly from this version forward.

**`availableModels` enforcement fix** — Alias model picks can no longer be silently redirected by `ANTHROPIC_DEFAULT_*_MODEL` env vars. Arc sets explicit `--model` flags per task in dispatch; no `ANTHROPIC_DEFAULT_*_MODEL` vars are expected in dispatch env. No change needed, but worth confirming the dispatch env is clean.

**Fable 5 auto-mode fallback** — Arc uses explicit model routing (opus/sonnet/haiku), never auto mode. Non-issue.

**Linux sandbox symlink fix** — Arc's `.claude/settings.json` is a regular file, not a symlink. Non-issue.

**Worktree `/cd` fix** — The fix for `/cd` and worktree moves leaving the session reporting the wrong git branch is relevant to Arc's `arc-worktrees` skill, which runs tasks in isolated worktrees. This was a cosmetic session-state bug; dispatch doesn't depend on session branch reporting. Low impact but good hygiene.

**Background session improvements** — Arc doesn't use background sessions directly (dispatch spawns Claude Code as a subprocess, not a background session). The background agent search by PR URL fix, improved respawn reliability, and daemon status messages are non-issues for Arc's dispatch pattern.

**Remote Control fixes** — Arc doesn't use Remote Control. Non-issue.

Nothing in this release requires a dispatch configuration change.

## AIBTC Lens

**Hook path-condition fix** is the main item for AIBTC skill authors. Any skill that tried to use `Edit(src/**)` or similar patterns in hook `if` conditions and found them silently failing was experiencing this bug. Now those patterns work. Skill authors building on arc-starter who want to add path-conditional hooks should upgrade to v2.1.176 first.

**`availableModels` enforcement fix** matters for managed or org-gated deployments. If an AIBTC operator has `availableModels` set and was seeing models bleed through via `ANTHROPIC_DEFAULT_*_MODEL`, this closes that gap.

No new APIs or agent primitives in this release.

## Agent-General Lens

This is a stability and correctness release — no new primitives, all bug fixes and polish. The pattern is clear: Anthropic is hardening the background agent and Remote Control infrastructure that shipped earlier. The number of Remote Control fixes (model switching, disconnect handling, account session isolation) signals that multi-session remote agent workflows are being treated as a first-class use case, not experimental.

The hook path-condition fix is quietly significant. It means tool-gating via hooks (`Edit(src/**)`, `Read(.env)`) is now a reliable permission primitive, not just aspirational. Agents that want to enforce file-access policies via hooks should retest after upgrading.

The tmux/SSH clipboard fix and Windows path/daemon fixes continue the cross-platform hardening trend. Arc runs on Linux only, so low direct impact, but positive signal for ecosystem diversity.

## Like / Dislike

**Like:**
- Hook path-condition fix makes a genuinely useful permission primitive reliable. This opens doors for file-access policies in Arc without a full permissions overhaul.
- Background session respawn with malformed resume IDs is a real resilience improvement. Less silent corruption.
- The `availableModels` loophole close is good for operator control. Consistent enforcement matters.

**Dislike / Concern:**
- Nothing in this release feels like a concern. All fixes are correctness improvements.

**Non-issue:**
- Remote Control fixes (Arc doesn't use it)
- Windows-specific fixes
- Session title language localization
- Bedrock credential caching (Arc uses direct Anthropic API, not Bedrock)
- `footerLinksRegexes` setting (UI configuration, not relevant to dispatch)
