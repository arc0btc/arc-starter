## Arc Lens

**Two actionable items; several passive improvements.**

### 1. `ENABLE_PROMPT_CACHING_1H` — High value for dispatch (P6)

The new env var opts into 1-hour prompt cache TTL instead of the default 5-minute TTL. This is directly relevant to Arc's dispatch pattern.

Arc loads the same static context every dispatch cycle: `SOUL.md`, `CLAUDE.md`, `MEMORY.md`, and skill `SKILL.md` files. With the default 5-minute TTL, that context cache expires between most cycles (dispatch fires up to every 30 minutes). With a 1-hour TTL, the static context stays cached across multiple cycles within the hour — input token costs for cached content drop by ~85% (cache read price vs. input price).

Estimated impact: 20-40% total cost reduction per dispatch cycle depending on context size.

To enable: set `ENABLE_PROMPT_CACHING_1H=1` in the dispatch service's environment (systemd unit `Environment=` or `.env` file loaded by the service). Requires v2.1.108+.

Note: `FORCE_PROMPT_CACHING_5M` forces 5-minute TTL explicitly — not relevant for Arc but useful if we ever need to debug caching behavior.

Also: a warning now shows at startup when `DISABLE_PROMPT_CACHING*` env vars are present. Good sanity signal — if we accidentally disable caching, we'll know immediately.

**Prior memory entry** `claude-code-prompt-caching [P8]` was tracking `--exclude-dynamic-system-prompt-sections` from v2.1.98 for 20-30% reduction. The 1H TTL is a separate, larger lever — they stack.

### 2. Fixed: Agent tool permission prompts in auto mode when safety classifier hits context window (P6 passive)

The bug: if a dispatched task used the Agent tool (subagent delegation) and the safety classifier's transcript exceeded its context window, Claude Code would unexpectedly prompt for permission even in auto/`--dangerously-skip-permissions` mode. This could have caused dispatch to hang waiting for input that never comes.

Arc delegates heavily via the Agent tool. If this bug ever triggered in a dispatch cycle, it would have been a silent hang → dispatch timeout → task failure. The fix in v2.1.108 removes this class of failure. No action needed — just a good passive improvement.

### 3. Rate limit vs. plan limit error message distinction

Error messages now distinguish server rate limits from plan usage limits. Arc's `dispatch-gate` currently treats consecutive failures uniformly. The improved error messages make it easier to tune gate logic to handle these two cases differently (rate limits are transient; plan limits may require longer backoff). Not urgent — existing logic handles both conservatively.

### 4. Fixed: `CLAUDE_ENV_FILE` ending with `#` comment causing no Bash output

If Arc's env file (loaded via `CLAUDE_ENV_FILE`) ended with a comment line, all Bash tool output would be silently suppressed. If any skill's setup adds trailing comments to an env file, this could explain mysterious empty-output failures. Passive fix — review env files if unexplained Bash silence occurs.

## AIBTC Lens

**One notable capability addition for skill authors.**

### Skill tool can now invoke built-in slash commands

The model can now discover and invoke built-in slash commands (`/init`, `/review`, `/security-review`) via the Skill tool. This means Arc (or any arc-starter-based agent) can programmatically trigger things like `/review` during dispatch without user interaction.

For AIBTC skill authors: if a skill workflow benefits from triggering a built-in review or security check as part of its execution, this is now possible via the Skill tool in the task context. Most skill workflows are CLI-driven (`arc skills run`) so this is edge-case, but it's a new composition primitive.

## Agent-General Lens

**Two architectural signals this release.**

### 1. Prompt cache TTL as a first-class configuration axis

By exposing `ENABLE_PROMPT_CACHING_1H` and `FORCE_PROMPT_CACHING_5M`, Anthropic is treating cache TTL as an explicit operational choice rather than a fixed implementation detail. For long-running agents with stable system prompts (the dominant autonomous agent pattern), 1-hour TTL is almost universally better. The direction is: agents should be managing their caching strategy deliberately, not relying on defaults.

### 2. Session re-entry UX (recap feature) hints at long-session agent patterns

The `/recap` feature and `CLAUDE_CODE_ENABLE_AWAY_SUMMARY` suggest Anthropic is building for long-lived sessions where humans step away and return. For autonomous agents (no human re-entry), this is a non-event. But the architectural implication is: Anthropic sees value in sessions that persist across attention gaps. Arc uses short-lived dispatch subprocesses — the opposite pattern. Worth watching if session persistence becomes a first-class primitive with persistent memory benefits.

### 3. Self-invoking slash commands via Skill tool = meta-programming primitive

Agents can now trigger `/security-review` or `/review` on their own work as a step in a larger workflow. This is the beginning of agents reasoning about their own code quality within a task. Early-stage, but the primitive is there.

## Like / Dislike

**Like:**
- `ENABLE_PROMPT_CACHING_1H`: long overdue. 5-minute TTL was nearly useless for non-interactive agent dispatch patterns. This change alone makes v2.1.108 worth upgrading to immediately.
- Agent tool permission fix in auto mode: eliminates a class of silent dispatch hangs that were hard to diagnose.
- Rate limit vs. plan limit error distinction: small but useful for operational monitoring.

**Dislike / Concern:**
- The recap feature (`CLAUDE_CODE_ENABLE_AWAY_SUMMARY`) requires telemetry to be enabled by default. Arc disables telemetry. The `CLAUDE_CODE_ENABLE_AWAY_SUMMARY` env var override exists, but it's one more "disabled by default for privacy-respecting setups" feature that requires explicit opt-in. Pattern continues.

**Non-issue:**
- `/undo` as `/rewind` alias — interactive UX, no impact.
- `/resume` picker defaulting to current directory — interactive only.
- Most UX polish (verbose indicator, diacritical marks fix, session title fixes) — all interactive, all irrelevant to headless dispatch.
