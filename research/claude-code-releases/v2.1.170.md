# Claude Code v2.1.170 — Applicability Assessment

*Published: 2026-06-09T17:23:09Z*
*URL: https://github.com/anthropics/claude-code/releases/tag/v2.1.170*
*Assessed: 2026-06-09*

## Release Notes (Full)

- Introducing **Claude Fable 5**: a Mythos-class model that Anthropic describes as exceeding the capabilities of any previously generally available model. Access requires updating to v2.1.170. Announcement: https://www.anthropic.com/news/claude-fable-5-mythos-5
- Fixed sessions not saving transcripts (and not appearing in `--resume`) when launched from the VS Code integrated terminal or any shell that inherited Claude Code environment variables.

---

## Arc Lens

**Claude Fable 5 is the dominant item in this release.** This is a new top-tier model — positioned above Opus 4.8, described as "Mythos-class" and surpassing all previously available models.

Arc's current `MODEL_IDS.opus` is set to `claude-opus-4-8` (commit 8d8b18a5, 2026-05-29). Fable 5 may supersede Opus for high-stakes dispatch tasks — but the concrete model ID is not yet confirmed in this release note. Before Arc can route tasks to Fable 5:

1. **Determine the model ID**: The release note does not state the API model string. It needs to be fetched from Anthropic docs or tested via `claude --version` output. Likely `claude-fable-5-*` but cannot be assumed.
2. **Evaluate cost/capability tradeoff**: If Fable 5 is priced above Opus 4.8, only specific task classes (RFC implementation, complex code review, multi-step architecture decisions) warrant the upgrade. Arc's current opus dispatch costs ~$0.471/task at C:2 — Fable costs could push this higher.
3. **Update MODEL_IDS** once the ID is confirmed: `src/dispatch.ts` or wherever `MODEL_IDS` is defined needs a `fable` key. Tasks with `model=fable` should route to the confirmed ID.

The VS Code terminal transcript fix does **not** apply to Arc. Arc's dispatch subprocess is launched by systemd, not from a VS Code integrated terminal. Arc already has full transcript/resume capability through its own dispatch cycle logging.

**Nothing breaks in this release for Arc.** The only action item is evaluating and potentially onboarding Fable 5.

## Agent-General Lens

**Anthropic is shipping a new capability tier.** "Fable" and "Mythos-class" appear to be branding for a model generation above the current Claude 4.x Opus tier. This signals:

- The model capability ceiling is moving. Agents that benchmark their "maximum intelligence" against Opus 4.8 should re-anchor against Fable 5 for critical tasks.
- The naming shift (Fable/Mythos) may indicate a deliberate move away from the Haiku/Sonnet/Opus naming hierarchy — worth watching whether this becomes a parallel product line or a full replacement.
- The "made safe for general use" framing implies Anthropic ran extended safety evaluation before release — this is consistent with their practice for their most capable models.

The transcript fix reveals that environment variable inheritance from VS Code terminals was silently causing session state loss. For agents that spawn Claude Code subprocesses in CI or IDE environments, this was a hidden footgun. The fix normalizes behavior across launch contexts.

## Like / Dislike

**Like:**
- Fable 5's availability is significant signal. If it genuinely exceeds Opus 4.8, Arc gains access to a materially more capable model for its hardest tasks (RFC implementation, complex audits, architectural decisions).
- The transcript/resume fix is clean and correct — VS Code environment variable inheritance was a valid bug, even if it doesn't affect Arc directly.

**Dislike / Concern:**
- The release note does not state the model ID. "Update to v2.1.170 for access" is not actionable for dispatch config without knowing the string to pass to `--model`. A follow-up to confirm the ID is required before any routing changes.
- "Mythos-class" and "Fable" branding without a clear positioning document creates ambiguity about whether this is Opus-tier++ or a separate product line. Arc's model routing (`opus`, `sonnet`, `haiku`) may need a new tier.

**Non-issue:**
- VS Code transcript fix — not relevant to Arc's systemd dispatch model.
